这个文档中的内容是按照我自己的面试经历，对面试中可能涉及的知识点进行的整理。但是所梳理的内容并不包含算法题，因为算法题对于校招同学来说要求会比较高，但对于社招来说要求会很低。如果你是校招生，建议去看其他人整理的算法题复习思路（看情况吧，以后我可能也会整理一下）；如果是社招，那么算法题着重看二叉树、链表、排序算法的简单题就够了。可以根据在 [codetop](https://codetop.cc/#/home) 上列出的算法题有针对性的进行复习。

本文档包含以下内容：

1. Java 基础：快速回顾一下 Java 基础知识点
2. Java 容器：面试必问，此部分内容要多看
3. Java 并发：面试必问，此部分内容要多看
4. Java I/O：待整理，基本不问
5. JVM：面试必问，此部分内容要多看
6. 框架使用：问的较少，不建议花太多时间准备，如果有微服务相关经验建议好好准备一下微服务相关的内容
7. MySQL：面试必问，此部分内容要多看
8. Redis：相对 MySQL 而言，问的会少一些
9. 中间件：问的相对较少，一般可能会在二面问问
10. 计算机网络：除了百度其他的都问的比较少
11. 操作系统：除了 I/O 多路复用，其他的都问的比较少
12. 设计思想：主要要会写单例，可能会考手写单例
13. 面经：面经整理
